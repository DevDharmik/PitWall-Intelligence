# PitWall Intelligence

> 75 seasons of Formula One, distilled into one interactive dashboard. Explore all-time race wins, world champions, constructor dynasties, and every circuit on the map — from Silverstone to Suzuka. Covering 1,125 grands prix, 861 drivers, and 212 teams across the full history of the sport, 1950–2024.

## Overview

PitWall Intelligence is a static, zero-dependency interactive dashboard built on the complete Ergast Formula One dataset. Every aggregation runs at build time via a Python ETL script — the browser receives a single pre-baked HTML file with all data embedded. No server. No API calls. No framework.

**Live demo:** `https://devdharmik.github.io/PitWall-Intelligence`

---

## Features

| Tab | What's Inside |
|---|---|
| **Overview** | Headline stats, top driver and constructor wins, races-per-season growth chart (1950–2024) |
| **Drivers** | Full top-20 all-time wins ranking, titles by nationality, multi-champion leaderboard, cumulative career wins chart, clickable season-by-season champion timeline |
| **Constructors** | Race wins and championship counts, constructor titles timeline (1958–2024), horizontal wins comparison chart |
| **Seasons** | Full season table — driver champion, nationality, points, constructor champion for every year |
| **Circuits** | Hand-drawn world map (Canvas API, equirectangular projection) with all 77 circuits plotted, full circuit directory |

## Architecture

```
data/raw/*.csv
      ↓  pandas ETL (scripts/build.py)
data/processed/f1_data.json   (~29KB)
      ↓  bundler (scripts/bundle.py)
dist/f1_dashboard.html         (single self-contained file)
      ↓
GitHub Pages → browser
```

**Runtime:** The browser gets one HTML file. Chart.js is loaded from a CDN. Everything else — DOM rendering, Canvas map, timeline interactions — is vanilla JavaScript operating on the embedded JSON.

## Getting Started

### Prerequisites

- Python 3.9+
- pip

### 1. Clone the repo

```bash
git clone https://github.com/DevDharmik/PitWall-Intelligence.git
cd PitWall-Intelligence
```

### 2. Install dependencies

```bash
pip install -r scripts/requirements.txt
```

`requirements.txt`:
```
pandas>=2.0
numpy>=1.24
```

### 3. Run the ETL

```bash
python scripts/build.py
```

This reads all 14 CSVs from `data/raw/`, runs every aggregation, and writes `data/processed/f1_data.json`.

### 4. Bundle into a single HTML file

```bash
python scripts/bundle.py
```

Output: `dist/f1_dashboard.html` — open this directly in any browser, no server needed.

### 5. (Optional) Run locally with live reload

```bash
cd src
python -m http.server 8000
# open http://localhost:8000
```

## Data Transformations

The build script (`scripts/build.py`) produces the following aggregations:

**Driver champions by year**
```
driver_standings + races → last raceId per year → position == 1
→ joined with drivers for name + nationality
```

**All-time race wins**
```
results (position == '1') → joined with drivers → groupby name → count
→ sorted descending, top 20
```

**Cumulative career wins (top 5 drivers)**
```
results + races + drivers → filter top 5 → groupby (name, year) → cumsum
→ one series per driver for the career progression chart
```

**Constructor champions by year**
```
constructor_standings + races → last raceId per year → position == 1
→ joined with constructors for name + nationality
```

**Circuit world map**
```
circuits.csv → lat/lng coordinates
→ equirectangular projection in Canvas 2D API
```

## CI/CD — GitHub Actions

On every push to `main` that touches `data/raw/`, `scripts/`, or `src/`, the workflow:

1. Sets up Python 3.11
2. Installs pandas
3. Runs `build.py` → produces `f1_data.json`
4. Runs `bundle.py` → produces `dist/f1_dashboard.html`
5. Deploys `dist/` to GitHub Pages

```yaml
# .github/workflows/build.yml
name: Build & Deploy

on:
  push:
    branches: [main]
    paths:
      - 'data/raw/**'
      - 'scripts/**'
      - 'src/**'

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'

      - run: pip install -r scripts/requirements.txt

      - run: python scripts/build.py

      - run: python scripts/bundle.py

      - uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./dist
```
